Here’s a student-friendly, not-too-dense guide to torch.optim.SGD, with plenty of small, runnable examples.

What is SGD in PyTorch?
- SGD = stochastic gradient descent. Each update moves model parameters a little toward reducing the loss, using gradients computed from a (usually small) batch.
- In PyTorch, you wrap the parameters you want to train in an optimizer: optimizer = torch.optim.SGD(params, lr=..., momentum=..., weight_decay=..., dampening=..., nesterov=...).
- Typical loop: zero_grad → forward pass → compute loss → backward() → optimizer.step() → repeat.

Key hyperparameters (what they do)
- lr (learning rate): how big each step is. Too large = unstable; too small = slow training.
- momentum: adds “memory” of past gradients. Helps smooth updates and move faster in flat valleys.
- dampening: reduces momentum on each step. Often left at 0.
- weight_decay: L2 regularization. Penalizes large weights to prevent overfitting.
- nesterov: a slightly different variant of momentum (true look-ahead). Usually combined with momentum > 0.
- maximize: not common; you can use it if you want to maximize a score instead of minimize loss.

How to use SGD in a simple loop (the skeleton)
- Create a model
- Create a loss function
- Create an optimizer with model.parameters()
- For each batch/epoch: zero_grad, compute loss, backward, optimizer.step

Tiny example 1: linear regression (y = 2x + 1 with noise)
Code:
import torch
from torch import nn
import torch.optim as optim

# Synthetic data: y = 2*x + 1 + noise
x = torch.randn(100, 1)
true_w = torch.tensor([[2.0]])
true_b = torch.tensor([1.0])
y = x @ true_w + true_b + 0.1 * torch.randn(100, 1)

model = nn.Linear(1, 1)  # learns a weight and bias
criterion = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.1, momentum=0.9)

for epoch in range(100):
    optimizer.zero_grad()          # reset gradients
    pred = model(x)                  # forward
    loss = criterion(pred, y)      # loss
    loss.backward()                  # backprop
    optimizer.step()                 # update parameters

    if epoch % 20 == 0:
        print(f"Epoch {epoch}, loss: {loss.item():.4f}")

What you should see: loss goes down as the model learns w ~ 2 and b ~ 1.

Tiny example 2: logistic regression (binary classification)
Code:
import torch
from torch import nn
import torch.optim as optim

# Simple binary dataset: two features per example
X = torch.randn(200, 2)
# Create a linear decision boundary with label 1 if sum>0, else 0
logits = X.sum(dim=1, keepdim=True)
y = (logits > 0).float()

model = nn.Linear(2, 1)  # logistic regression
criterion = nn.BCEWithLogitsLoss()  # combines sigmoid + BCE
optimizer = optim.SGD(model.parameters(), lr=0.1, momentum=0.9)

for epoch in range(200):
    optimizer.zero_grad()
    logits_out = model(X)                          # forward
    loss = criterion(logits_out, y)               # loss
    loss.backward()                                 # backprop
    optimizer.step()

    if epoch % 40 == 0:
        with torch.no_grad():
            pred = (logits_out > 0).float()
            acc = (pred == y).float().mean().item()
        print(f"Epoch {epoch}, loss: {loss.item():.4f}, acc: {acc:.2f}")

Tiny example 3: a small neural net to show SGD on more layers
Code:
import torch
from torch import nn
import torch.optim as optim

# Fake dataset: simple multiclass with 3 classes
X = torch.randn(300, 4)
y = torch.randint(0, 3, (300,))

class TinyNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(4, 10),
            nn.ReLU(),
            nn.Linear(10, 3)
        )
    def forward(self, x):
        return self.net(x)

model = TinyNet()
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(model.parameters(), lr=0.05, momentum=0.9)

for epoch in range(100):
    optimizer.zero_grad()
    outputs = model(X)
    loss = criterion(outputs, y)
    loss.backward()
    optimizer.step()

    if epoch % 20 == 0:
        _, pred = outputs.max(1)
        acc = (pred == y).float().mean().item()
        print(f"Epoch {epoch}, loss: {loss.item():.4f}, acc: {acc:.2f}")

import torch

model = nn.Linear(5, 2)
criterion = nn.MSELoss()
# Add weight_decay to penalize large weights
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9, weight_decay=0.01)

# pretend data
X = torch.randn(60, 5)
y = torch.randn(60, 2)

for epoch in range(60):
    optimizer.zero_grad()
    pred = model(X)
    loss = criterion(pred, y)
    loss.backward()
    optimizer.step()

Practical tips to keep SGD simple
- Always zero gradients before a new step: optimizer.zero_grad() (or model.zero_grad()).
- Keep batches small enough to be fast, but large enough to give useful gradients.
- Start with a sensible lr (e.g., 0.1 for small problems, 0.01-0.1 for many nets) and adjust.
- Use momentum if you want smoother, faster convergence in many problems.
- If overfitting, try weight_decay (L2 regularization) or add dropout (outside SGD wrapper, in model).
- Monitor loss (and accuracy for classifiers) on a validation set to avoid overfitting and to tune lr/momentum.
- If training is unstable (loss spikes or NaNs), reduce lr, or turn off momentum temporarily.

Common pitfalls
- Forgetting to call optimizer.zero_grad() leads to gradient accumulation and wrong updates.
- Using the wrong device (CPU vs GPU) without moving model/data.
- Forgetting that loss.backward() accumulates gradients; you should zero them if you’re not stepping every batch.

TL;DR
- SGD updates model parameters using gradients from a batch.
- Tune lr, momentum, weight_decay to trade off speed, stability, and regularization.
- Typical loop: zero_grad → forward → loss → backward → step.
- You can try simple linear models, then small MLPs, tweaking hyperparameters as you go.
- Use the provided examples as templates to build your own training scripts.
