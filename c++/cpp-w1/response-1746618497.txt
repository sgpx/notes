# Recommended C++ Practice Problems (Repetitive, Relevant, Skill-building)

### 1. **Vector & Memory Management Loop**
- Write a function that accepts a `std::vector<int>&` and removes all even numbers in-place using `erase-remove_if`.
- Now, repeat with `std::vector<std::unique_ptr<int>>` (unique_ptr, move semantics required).

---

### 2. **Rebuild the StringWrapper: With Copy/Move Logging**
- Redo your `StringWrapper` class, but *log every copy and move* (print a message inside each special member function).
- Write code that creates, copies, and moves several StringWrapper objects (`std::vector`, `std::sort`, etc.) so you can observe the logs.  
- Repeat with `std::shared_ptr<StringWrapper>` and track what happens.

---

### 3. **Template: Type Conversion Add**
- Write a function template `add(a, b)` that works if `a` and `b` are *different* types (e.g. `int` and `double`) by returning the correct type.
- Test with various type pairs: `add(1, 2.5)`, `add('a', 3)`.

---

### 4. **Smart Pointer Polymorphism**
- Create a hierarchy: base class `Animal` (pure virtual `speak`), derived `Dog`/`Cat`.
- Store them as `std::unique_ptr<Animal>` in a `std::vector`, iterate and call `speak()` for each.
- Replace with `std::shared_ptr<Animal>`, repeat (ensure correct destruction).

---

### 5. **Resource Management in Loops (RAII)**
- Write a class that manages an array of integers (manual `new[]`/`delete[]`).
- Instantiate this in a loop (`std::vector` of your objects), tracking allocations and deallocations.
- Replace with `std::vector<int>`, compare memory safety.

---

### 6. **Variadic Debug Challenge (Multiple Containers)**
- Extend your `debug()` variadic template to work with both single values and arbitrary containers (print elements if parameter is a container, else print the value).
- Test with integers, vectors, arrays, and mixtures.

---

### 7. **STL Algorithms—Practice Rewrites**
- For a given vector, write manual code to:
    - Find the minimum element (not with `std::min_element`)
    - Sort the vector (implement bubble/selection sort by hand)
    - Reverse the vector
- Then, repeat the above using only STL algorithms.

---

### 8. **Function Object (Functor) Practice**
- Create a functor (class with `operator()`) to filter and store just odd numbers from a list to a new vector.
- Replace with lambda and `std::function`, compare code and output.

---

### 9. **Copy Elision and Return Value Optimization (RVO)**
- Write a function that returns a (big) user class by value.
- Print inside constructors/destructors to observe when/how many moves/copies happen.
- Try with and without `std::move`, see what changes.

---

### 10. **Constructor Delegation**
- Create a class with several constructors using *constructor delegation* (i.e., one constructor calls another).
- Test construction with different argument counts/types, print which constructor is called.

---

### 11. **Sorting Objects with Custom Comparator**
- Define a vector of custom objects (e.g., `Person` with `name` and `age`).
-** Sort by `name`, then by `age` using custom comparators (lambdas and function objects)**.
- Repeat with `std::function`-based comparator.

---

## Bonus: Deeper/Remix Concepts

### 12. **Rule of Five (Class with all Special Functions)**
- Create a class that manages a resource (like a C array or file handle).
- Implement: destructor, copy constructor, move constructor, copy assignment, move assignment.
- Write code that exercises all of these, and logs when they're called.

---

### 13. **Observer Pattern with smart pointers**
- Implement a basic observer/listener pattern.
- Store observers as `std::weak_ptr` and subjects as `std::shared_ptr`.
- Practice registration/deregistration and safe notification.

---

## How to Practice for Best Retention:
- **Repeat** each problem with variations (change types, containers, use different STL features).
- **Mix containers**, try `list`, `deque`, etc., not just `vector`.
- **Try writing tests/cases that deliberately break your code** (e.g., double frees, use-after-move).
