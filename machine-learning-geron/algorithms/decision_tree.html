2. The Decision Tree Model

Based on the table above, the algorithm builds this tree to make future predictions.

A. Why Each Split Happens (Inference During Training)

  • Root split (Outlook): The algorithm checks candidate features and picks the one that makes the child groups most pure. Outlook wins because the Overcast branch becomes 100% "Yes".
  • Sunny branch split (Humidity): For Sunny days, Humidity perfectly separates outcomes: High → No, Normal → Yes.
  • Rain branch split (Wind): For Rain days, Wind perfectly separates outcomes: Strong → No, Weak → Yes.

Why there is no Overcast sub-branch: All Overcast rows are already “Yes,” so the node is perfectly pure. The tree stops there and makes a leaf instead of splitting further.

Why Temperature isn’t used: When the algorithm tested Temperature at each node, it didn’t improve purity as much as Outlook, Humidity, or Wind. Since other features produced cleaner splits, Temperature was never selected for a split.

Root split math (Gini for all features):

  • Parent: Yes = 9, No = 5 → Gini = 1 − (9/14)2 − (5/14)2 = 0.459
  • Outlook: Sunny (2Y,3N) Gini = 0.48; Overcast (4Y,0N) Gini = 0; Rain (3Y,2N) Gini = 0.48 → Weighted = (5/14)·0.48 + (4/14)·0 + (5/14)·0.48 = 0.343
  • Temperature: Hot (2Y,2N) Gini = 0.50; Mild (4Y,2N) Gini = 0.444; Cool (3Y,1N) Gini = 0.375 → Weighted = (4/14)·0.50 + (6/14)·0.444 + (4/14)·0.375 = 0.441
  • Humidity: High (3Y,4N) Gini = 0.490; Normal (6Y,1N) Gini = 0.245 → Weighted = (7/14)·0.490 + (7/14)·0.245 = 0.367
  • Wind: Weak (6Y,2N) Gini = 0.375; Strong (3Y,3N) Gini = 0.50 → Weighted = (8/14)·0.375 + (6/14)·0.50 = 0.429

Conclusion: Outlook has the lowest weighted Gini (0.343), so it is selected at the root. Temperature is higher (0.441), so it is not chosen.

B. Inference at Prediction Time (How a New Row Is Classified)

At prediction time, the model does not recompute splits. It simply follows the finished tree from root to leaf using the new row’s feature values.

  • Step 1: Start at the root (Outlook).
  • Step 2: Follow the branch that matches the row’s value.
  • Step 3: Repeat until a leaf node is reached.
  • Step 4: The leaf’s class (and optional probability) is the prediction.

C. Final Tree as Rules (Readable Inference)

  • Rule 1: IF Outlook = Overcast → Play Tennis = Yes.
  • Rule 2: IF Outlook = Sunny AND Humidity = High → Play Tennis = No.
  • Rule 3: IF Outlook = Sunny AND Humidity = Normal → Play Tennis = Yes.
  • Rule 4: IF Outlook = Rain AND Wind = Strong → Play Tennis = No.
  • Rule 5: IF Outlook = Rain AND Wind = Weak → Play Tennis = Yes.

D. Worked Example (Inference on a New Day)

Scenario: A new day arrives and we want to predict whether people will play tennis.

New row (unseen): Outlook = Sunny, Temperature = Mild, Humidity = High, Wind = Weak

  • Step 1 (Root): Outlook = Sunny → follow the Sunny branch to the Humidity node.
  • Step 2: Humidity = High → follow the High branch to the leaf.
  • Leaf output: Play Tennis = No.
  • Final prediction: No.

E. Training Notes (Gini + Pruning + Interpretation)

  • Gini impurity: Gini = 1 − Σ(pk2). The chosen split minimizes the weighted child Gini.
  • Information gain (optional): Gain = H(parent) − weighted child entropy.
  • Pre-pruning: Stop early using `max_depth`, `min_samples_split`, `min_samples_leaf`, or `min_impurity_decrease`.
  • Post-pruning: Remove weak branches using validation data or cost-complexity pruning.
  • Interpretation: Each root-to-leaf path is a human-readable rule.

F. For Dummies: Full Step-by-Step (Handholding) - Tennis Example

Goal: We want our decision tree to predict if people will Play Tennis (Yes/No) based on the weather (Outlook, Temperature, Humidity, Wind).

Imagine you are the algorithm: You have to pick one weather column to ask about first. You want to pick the question that separates the "Yes" days from the "No" days the most clearly.

Step 1: Start with all 14 days.

Right now, we have 9 "Yes" days and 5 "No" days. It's a messy, mixed-up group. The algorithm calculates a "messiness score" (called Gini Impurity). Let's see how that's calculated:

The Math Behind the "Messiness": Gini Impurity

The formula for finding the Gini Impurity of a single group is:

$$ \text{Gini} = 1 - \sum (p_i)^2 $$

Where \( p_i \) is the probability of class \( i \). For Tennis, types are \( \text{"Yes"} \) and \( \text{"No"} \).

1. Starting "Messiness" (Parent Node):

Starting with \( 9 \) Yes and \( 5 \) No (Total \( 14 \)):

$$ \text{Parent Gini} = 1 - \left(\frac{9}{14}\right)^2 - \left(\frac{5}{14}\right)^2 \approx \mathbf{0.459} $$

2. Evaluating "Outlook" Split:

  • Sunny: \( 2 \)Y, \( 3 \)N \(\rightarrow \text{Gini} = 1 - (\frac{2}{5})^2 - (\frac{3}{5})^2 = 0.48 \)
  • Overcast: \( 4 \)Y, \( 0 \)N \(\rightarrow \text{Gini} = 1 - (\frac{4}{4})^2 - (\frac{0}{4})^2 = 0 \)
  • Rain: \( 3 \)Y, \( 2 \)N \(\rightarrow \text{Gini} = 1 - (\frac{3}{5})^2 - (\frac{2}{5})^2 = 0.48 \)

The Weighted Average messiness for Outlook is:

$$ \text{Outlook Weighted Gini} = \frac{5}{14}(0.48) + \frac{4}{14}(0) + \frac{5}{14}(0.48) \approx \mathbf{0.343} $$

3. Evaluating "Temperature" Split:

  • Hot: \( 2 \)Y, \( 2 \)N \(\rightarrow \text{Gini} = 1 - (\frac{2}{4})^2 - (\frac{2}{4})^2 = 0.50 \)
  • Mild: \( 4 \)Y, \( 2 \)N \(\rightarrow \text{Gini} = 1 - (\frac{4}{6})^2 - (\frac{2}{6})^2 \approx 0.444 \)
  • Cool: \( 3 \)Y, \( 1 \)N \(\rightarrow \text{Gini} = 1 - (\frac{3}{4})^2 - (\frac{1}{4})^2 = 0.375 \)

The Weighted Average messiness for Temperature is:

$$ \text{Temp Weighted Gini} = \frac{4}{14}(0.50) + \frac{6}{14}(0.444) + \frac{4}{14}(0.375) \approx \mathbf{0.441} $$

Conclusion: The algorithm compares the scores. Since \( 0.343 \) (Outlook) is lower than \( 0.441 \) (Temperature), the group became "less messy" with Outlook. Outlook wins and becomes the root node!

Step 2: Outlook wins as the Root Node.

The algorithm compared all features as shown above, and Outlook produced the largest reduction in "messiness." So, Outlook becomes our first question.

Step 3: Process the Branches.

  • If Outlook is Overcast: All days are "Yes". This group is perfectly pure, so the algorithm stops here and makes a "Yes" leaf.
  • If Outlook is Sunny: Mixed (2 Yes, 3 No). We need a new question for these 5 days!
  • If Outlook is Rain: Mixed (3 Yes, 2 No). We need a new question for these 5 days!

Step 4: Drill down into the Sunny branch.

Looking only at the 5 Sunny days, the algorithm tests the remaining columns (Humidity, Wind, Temperature). Since Humidity perfectly separates them (High = No, Normal = Yes) with a Gini score of \( 0 \), it wins the branch.

Step 5: Drill down into the Rain branch.

Looking only at the 5 Rain days, the algorithm finds that Wind perfectly separates them (Strong = No, Weak = Yes) with a Gini score of \( 0 \), so it becomes the final split.

Frequently Asked Questions (FAQ)

  • Why was the "Temperature" column completely ignored in the final tree? Because at every single step where the algorithm had to pick a split, another column did a better job at separating Yes from No. At the root, Outlook was better. In the Sunny branch, Humidity was better. In the Rain branch, Wind was better. Temperature never won a single "messiness" contest, so it never got added to the tree!
  • Why isn't "Outlook" asked again further down the tree? Once we split by a column, all rows in that branch have the exact same value for it. Every row in the Sunny branch is Sunny. Asking "Is it Sunny?" again wouldn't separate the data any further—it's redundant.
  • What if a column doesn't split the data perfectly? (What if a group is still messy?) That's completely normal! If a sub-group is still mixed (like the 5 "Sunny" days that had 2 Yes and 3 No), the algorithm just repeats the process inside that specific branch. It will test the remaining columns (like Humidity or Wind) only on those 5 days to find a new question that separates them further. It keeps drilling down until the groups become perfectly pure, or it runs out of questions!
  • Why does an impurity score of 0 mean the tree stops growing (no more splits)? Two reasons:
      1. Logically: We already have the answer! If a group is "4 Yes and 0 No" (Impurity = 0), we are 100% certain the answer is "Yes". Asking another question like "Was it windy?" won't change our certainty.
      2. Mathematically: The algorithm's goal is to reduce the messiness score (this reduction is called Information Gain). Impurity can't be lower than 0. If we split a pure "Yes" group by Wind, we might get "2 Yes" (Weak wind) and "2 Yes" (Strong wind). Both new groups still have an impurity of 0. Since the score didn't drop (0 to 0), the algorithm says "This split provides zero new information" and turns the node into a final "Leaf" instead.

How the Tree Is Built (Step-by-Step, Multi-Class)

Step 1: Compute parent Gini.

Parent class counts: Tech = 4, Home = 6, Grocery = 4 (total = 14)

Gini(parent) = 1 − (4/14)2 − (6/14)2 − (4/14)2 = 0.653

Step 2: Evaluate candidate splits. We try each feature and compute the weighted Gini after the split.

Example split on Age:

  • Youth: Tech = 4, Home = 1, Grocery = 0 → Gini = 1 − (4/5)2 − (1/5)2 = 0.32
  • Middle: Home = 4 → Gini = 0
  • Senior: Grocery = 4, Home = 1 → Gini = 0.32

Weighted Gini(Age) = (5/14)·0.32 + (4/14)·0 + (5/14)·0.32 = 0.229

The algorithm computes the same for other features (Income, Student, Credit Rating) and chooses the lowest weighted Gini. Assume Age is best, so Age becomes the root.

Step 3: Split each branch and repeat.

  • Middle branch: All are Home → pure leaf (stop).
  • Youth branch: Try splits again. Student works well: Student = No → (Tech, Tech, Tech) Gini = 0; Student = Yes → (Tech, Home) Gini = 0.5; weighted = 0.2 → choose Student.
  • Senior branch: Credit Rating works well: Fair → (Grocery, Grocery, Grocery) Gini = 0; Excellent → (Home, Grocery) Gini = 0.5; weighted = 0.2 → choose Credit Rating.

Step 4: Stop rules. When a node is pure or hits limits like max depth or min samples, it becomes a leaf with the majority class.

For Dummies: Full Step-by-Step (Handholding)

Goal: Build a tree that predicts the Purchase Category (Tech/Home/Grocery) from the columns.

Think of it like this: Each split asks a simple question (a column), and we want questions that separate the classes as cleanly as possible.

Step 1: Start with the whole dataset.

You have 14 rows with 3 classes. Counts are Tech = 4, Home = 6, Grocery = 4. This mix is “impure.”

Step 2: Try each column as a first split. You test Age, Income, Student, Credit Rating.

For each column, you:

  • Group the rows by that column’s values (e.g., Age → Youth/Middle/Senior).
  • Compute the Gini impurity for each group (lower is better).
  • Compute the weighted average Gini across groups.

Step 3: Pick the “best splitter.”

The best splitter is the column with the lowest weighted Gini. Here are the weighted Gini values for each first split:

  • Age: 0.229 (computed above)
  • Income: 0.616
  • Student: 0.521
  • Credit Rating: 0.636

Result: Age is lowest, so Age becomes the root.

Step 4: Create the root node.

Root = “Age?” with 3 branches: Youth, Middle, Senior.

Step 5: Process each branch.

  • Middle: All rows are Home → pure → make a leaf “Home”.
  • Youth: Mixed (Tech/Home) → not pure → keep splitting.
  • Senior: Mixed (Grocery/Home) → not pure → keep splitting.

Step 6: Repeat the same split search inside each mixed branch.

For Youth rows, test remaining columns (Income, Student, Credit Rating). Student splits them best:

  • Youth + Student = No → all Tech → leaf “Tech”.
  • Youth + Student = Yes → mixed (Tech/Home) → could split again or stop if rule says so.

For Senior rows, Credit Rating splits best:

  • Senior + Fair → all Grocery → leaf “Grocery”.
  • Senior + Excellent → mixed (Home/Grocery) → could split again or stop by rule.

Step 7: Stop when a node is pure or you hit a rule.

Common stop rules: max depth, min samples, or no improvement. When you stop, you make a leaf with the majority class.

Final result: A tree of simple questions that routes any new row to a leaf (Tech/Home/Grocery).
