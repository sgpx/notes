Sure! Here's a medium-difficulty learning problem that can be solved **only using `torch.rand()`**, i.e., where you generate random inputs and outputs using `torch.rand()` (no `.normal()`, no `.randn()`, no other distributions), and still can train a simple model to solve it.

---

## Problem: Learning a Continuous Target Function from Random Inputs in [0,1]

**Task:**  
Given a dataset of input-output pairs \((x, y)\) where \(x \in [0,1]\) (a scalar) and

\[
y = \sin(2 \pi x) + 0.1 \cdot \text{noise}
\]

where noise is a uniform random noise sampled **only** from `torch.rand()` (scaled to \([-0.1, 0.1]\)), learn a small neural network to approximate \(\sin(2 \pi x)\).

---

### Why this is solvable only with `torch.rand()`:
- Inputs \(x\) are generated using `torch.rand()`, uniform on [0, 1].
- Noise is generated by using `torch.rand()` and scaled/shifted to uniform noise in a small range.
- The target \(y\) is noisy, continuous.
- Since inputs and noise come exclusively from `torch.rand()`, you are **not allowed** to use, say, `torch.randn()` or any Gaussian noise.

### Solution outline:
- Generate, say, 10,000 samples of inputs via `x = torch.rand(N, 1)`
- Generate noise as `noise = (torch.rand(N, 1) - 0.5) * 0.2` to get uniform noise in \([-0.1, 0.1]\)
- Outputs `y = torch.sin(2 * pi * x) + noise`
- Fit a small MLP network to learn `y` from `x`
- The network can be linear layers with `ReLU` or `Tanh` activations, trained with MSE loss.

---

## Code Example:

```python
import torch
import torch.nn as nn
import torch.optim as optim
import math
import matplotlib.pyplot as plt

# Fix seed for reproducibility
torch.manual_seed(0)

# Training data size
N = 10000

# Generate input data in [0,1]
x = torch.rand(N, 1)

# Generate uniform noise in [-0.1, 0.1] using torch.rand()
noise = (torch.rand(N, 1) - 0.5) * 0.2

# Target function with noise
y = torch.sin(2 * math.pi * x) + noise

# Define a small MLP model
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(1, 64),
            nn.Tanh(),
            nn.Linear(64, 64),
            nn.Tanh(),
            nn.Linear(64, 1)
        )

    def forward(self, x):
        return self.net(x)

model = MLP()

# Optimizer and loss
optimizer = optim.Adam(model.parameters(), lr=0.01)
loss_fn = nn.MSELoss()

# Train loop
epochs = 500
for epoch in range(epochs):
    optimizer.zero_grad()
    y_pred = model(x)
    loss = loss_fn(y_pred, y)
    loss.backward()
    optimizer.step()

    if (epoch+1) % 50 == 0:
        print(f"Epoch {epoch+1}/{epochs} - Loss: {loss.item():.6f}")

# Plot results (test with new data)
x_test = torch.linspace(0, 1, 100).unsqueeze(1)
with torch.no_grad():
    y_test_pred = model(x_test)

plt.plot(x_test.numpy(), y_test_pred.numpy(), label="Predicted")
plt.plot(x_test.numpy(), torch.sin(2 * math.pi * x_test).numpy(), label="True (sin)")
plt.legend()
plt.show()
```

---

### Explanation:
- The model learns to approximate the sine wave from noisy uniform samples.
- The noise and inputs come exclusively from `torch.rand()`.
- The function is smooth and periodic, so learning it is a medium complexity nonlinear regression problem.
- You cannot use normal distributions or other random generators — constraining the randomness to uniform only makes it more interesting.
