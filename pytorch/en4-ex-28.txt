Here’s a simple, student-friendly guide to torch.nn.ReLU with lots of bite-sized examples.

What is ReLU (in PyTorch)
- ReLU stands for Rectified Linear Unit.
- It’s an activation: y = max(0, x) applied to each element.
- PyTorch provides two ways to use it:
  - As a module: nn.ReLU()
  - As a function: F.relu(...) or torch.relu(...)

Why use ReLU
- Easy to compute.
- Helps neural nets learn by keeping positive signals and bottling down negatives.
- Gradients flow well for positive inputs (gradient is 1 there).

Tiny definitions you’ll hear
- Forward: y = max(0, x) for every element.
- Backward (gradient): dy/dx = 1 if x > 0, else 0 (x = 0 is a convention; PyTorch uses 0 there).

Short, sweet examples

1) Basic forward on a 1D tensor
- What you’ll see: negatives become zero, positives stay the same.

import torch
from torch import nn

x = torch.tensor([-1.5, 0.0, 2.0, 3.5])
relu = nn.ReLU()
y = relu(x)
print(y)  # tensor([0.0, 0.0, 2.0, 3.5])

2) 2D/tensor with batches (like a mini image or feature map)
- ReLU works element-wise.

x = torch.tensor([[-1.0, 2.0], [0.0, -3.0]])
print(nn.ReLU()(x))
# tensor([[0.0, 2.0],
#         [0.0, 0.0]])

3) ReLU inside a small neural net (Sequential)
- Common pattern: Linear -> ReLU -> Linear -> ...

model = torch.nn.Sequential(
    torch.nn.Linear(4, 3),
    torch.nn.ReLU(),
    torch.nn.Linear(3, 2)
)

inp = torch.randn(5, 4)  # batch of 5
out = model(inp)
print(out.shape)  # torch.Size([5, 2])

4) In-place ReLU (use with care)
- In-place saves memory but can complicate autograd if you reuse the input.

x = torch.tensor([-1.0, 0.5, -0.2], requires_grad=True)
relu_inplace = nn.ReLU(inplace=True)
y = relu_inplace(x)
print(y)  # tensor([0.0, 0.5, 0.0])

loss = y.sum()
loss.backward()
print(x.grad)  # tensor([0.0, 1.0, 0.0])

Tips about inplace:
- It overwrites x. If you need x later, don’t use inplace=True.
- For training, it’s often safer to use the non-inplace version.

5) ReLU vs functional ReLU (same math, different style)
- ReLU module keeps a small “state” (not much here, but useful in Sequential).
- Functional form is handy for quick calls.

import torch.nn.functional as F

x = torch.tensor([-2.0, 0.0, 3.0])
print(F.relu(x))        # tensor([0.0, 0.0, 3.0])
print(nn.ReLU()(x))     # also tensor([0.0, 0.0, 3.0])

6) Backprop test: gradients through ReLU
- Important to see how gradients flow.

x = torch.tensor([-1.0, 0.5, 3.0], requires_grad=True)
out = nn.ReLU()(x).sum()
out.backward()
print(x.grad)  # tensor([0., 1., 1.])

What to remember when you’re building nets
- Use after linear layers to inject non-linearity.
- Don’t overthink the exact numbers; just know:
  - Negative inputs become 0.
  - Positive inputs pass through.
  - Gradients flow only from positives (0 for negatives).
- In multi-layer nets, stacking ReLUs is common (e.g., Linear -> ReLU -> Linear -> ReLU).

Common gotchas (quick tips)
- In-place can save memory but may break gradient tracking if you reuse the input later.
- ReLU leaves zeros for negative inputs; you may see “dead neurons” if many inputs stay negative. LeakyReLU is a variant if this is a concern.
- If you’re debugging, try both nn.ReLU() and F.relu(...) to confirm behavior is as expected.
