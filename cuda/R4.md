### 1. SAXPY (Scalar Alpha X Plus Y)
**Concept:** **Read-only cache / Constant Memory intuition.** Introduces a uniform scalar broadcast to all threads. Precursor to understanding `__constant__` memory or kernel arguments passed by value.
```cpp
__global__ void saxpy(float *x, float *y, float a, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) y[i] = a * x[i] + y[i]; // y = a*x + y
}
// Host: Pass 'a' by value (e.g., 2.0f). No extra device malloc needed for scalar.
```

### 2. Vector Scale & Clamp (Element-wise Unary Op + Branching)
**Concept:** **Warp Divergence.** Threads in a warp take different paths (`if/else`). Shows that divergence serializes execution within a warp.
```cpp
__global__ void scale_clamp(float *data, float scale, float min_v, float max_v, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        float val = data[i] * scale;
        // Divergence happens here based on data values
        data[i] = (val < min_v) ? min_v : (val > max_v) ? max_v : val;
    }
}
```

### 3. 2D Grid / 2D Block (Image/Matrix Indexing)
**Concept:** **Multi-dimensional Mapping.** Essential for images, matrices, and stencils. Teaches `blockIdx.y`, `threadIdx.y`, and pitch-linear indexing.
```cpp
__global__ void transpose_2d(float *in, float *out, int width, int height) {
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    
    if (row < height && col < width) {
        // Input: row-major [row * width + col]
        // Output: transposed [col * height + row]
        out[col * height + row] = in[row * width + col];
    }
}
// Host: dim3 block(16, 16); dim3 grid((W+15)/16, (H+15)/16);
```

### 4. Grid-Stride Loop (Persistent Threads)
**Concept:** **Grid Size Independence / Occupancy.** Decouples problem size ($N$) from hardware limits (max grid size). Teaches the standard "production" pattern for large $N$.
```cpp
__global__ void vector_add_grid_stride(float *a, float *b, float *c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    int stride = gridDim.x * blockDim.x; // Total threads launched
    
    for (int i = idx; i < n; i += stride) { // Loop until done
        c[i] = a[i] + b[i];
    }
}
// Host: Launch fixed grid (e.g., 100 blocks * 256 threads) regardless of N.
```

### 5. Reduction (Sum) - Single Block (Shared Memory Intro)
**Concept:** **Inter-thread Communication & Synchronization (`__syncthreads()`).** The "Hello World" of shared memory. **Constraint:** Only works for $N \le 1024$ (single block).
```cpp
__global__ void block_sum_reduce(float *input, float *output, int n) {
    extern __shared__ float sdata[]; // Dynamic shared memory
    int tid = threadIdx.x;
    int i = blockIdx.x * blockDim.x + tid;

    // Load
    sdata[tid] = (i < n) ? input[i] : 0.0f;
    __syncthreads();

    // Reduce in shared mem (tree reduction)
    for (int s = blockDim.x / 2; s > 0; s >>= 1) {
        if (tid < s) sdata[tid] += sdata[tid + s];
        __syncthreads();
    }
    if (tid == 0) output[blockIdx.x] = sdata[0]; // Write partial sum
}
// Host: <<<1, 256, 256*sizeof(float)>>> (3rd arg = shared mem bytes)
```

### 6. Stencil / 1D Convolution (Halo/Neighbor Access)
**Concept:** **Memory Access Patterns & Boundary Conditions.** Reading neighbors (`i-1`, `i+1`). Introduces "Ghost Cells" or boundary checks.
```cpp
__global__ void stencil_1d(float *in, float *out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    // 3-point smoothing: out[i] = (in[i-1] + in[i] + in[i+1]) / 3
    if (i > 0 && i < n - 1) { // Interior points only (simple boundary)
        out[i] = (in[i-1] + in[i] + in[i+1]) * 0.3333f;
    } else if (i < n) { // Boundaries
        out[i] = in[i]; 
    }
}
```

### 7. Struct of Arrays (SoA) vs Array of Structs (AoS)
**Concept:** **Memory Coalescing.** The single most important performance concept for beginners. Compare bandwidth of two kernels.
```cpp
// AoS: Bad coalescing (strided access)
struct Particle { float x, y, z, mass; };
__global__ void aos_kernel(Particle *p, int n) { ... p[i].x ... }

// SoA: Perfect coalescing (contiguous access)
__global__ void soa_kernel(float *x, float *y, float *z, float *mass, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) x[i] += 1.0f; // Contiguous 128-byte transaction
}
```

### 8. Stream Compaction (Filter) - Two Pass
**Concept:** **Parallel Prefix Sum (Scan) Logic.** Foundation for "remove_if" in parallel. 
*Pass 1:* Predicate $\to$ Flags (1/0). 
*Pass 2:* Exclusive Scan on Flags $\to$ Indices. 
*Pass 3:* Scatter.
```cpp
// Simplified Pass 1: Generate Flags
__global__ void gen_flags(int *in, int *flags, int n, int threshold) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) flags[i] = (in[i] > threshold) ? 1 : 0;
}
// Student implements Scan (Pass 2) and Scatter (Pass 3) as follow-up exercises.
```

### 9. Histogram (Atomic Operations)
**Concept:** **Atomics & Contention.** Global memory atomics (`atomicAdd`). Shows serialization cost when many threads hit the same bin.
```cpp
__global__ void histogram(int *data, unsigned int *bins, int n, int num_bins) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        int bin = data[i] % num_bins; // Simple hash
        atomicAdd(&bins[bin], 1);     // Serializes if bin collision
    }
}
// Host: cudaMemset(bins, 0, num_bins * sizeof(int)) before launch.
```

### 10. Monte Carlo Pi Estimation (Random Numbers + Reduction)
**Concept:** **Curand / Random State + Reduction.** Embarrassingly parallel workload. Teaches `curand_init` (requires `curand_kernel.h`) and combining random gen with the reduction pattern from #5.
```cpp
#include <curand_kernel.h>

__global__ void monte_carlo_pi(float *partial_sums, int samples_per_thread, int seed) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    curandState state;
    curand_init(seed, tid, 0, &state); // Unique sequence per thread

    int inside = 0;
    for (int i = 0; i < samples_per_thread; i++) {
        float x = curand_uniform(&state);
        float y = curand_uniform(&state);
        if (x*x + y*y <= 1.0f) inside++;
    }
    // Use Block Reduction (from Ex 5) to sum 'inside' per block
    // Write block result to partial_sums[blockIdx.x]
}
// Host: Sum partial_sums on CPU -> Pi = 4.0 * total_inside / total_samples
```

### Pedagogical Progression Guide

| Exercise | Core Lesson | Key CUDA Keyword/Feature |
| :--- | :--- | :--- |
| **1. SAXPY** | Kernel Arguments / Broadcast | Pass-by-value args |
| **2. Clamp** | Warp Divergence | `if/else` in device code |
| **3. 2D Transpose** | Multi-dim Indexing | `dim3`, `threadIdx.y` |
| **4. Grid Stride** | Scalability / Occupancy | `gridDim.x`, `for` loop |
| **5. Block Reduce** | **Shared Memory / Sync** | `extern __shared__`, `__syncthreads()` |
| **6. Stencil** | Neighbor Access / Bounds | Boundary checks |
| **7. SoA vs AoS** | **Coalescing (Performance)** | Memory Layout |
| **8. Compaction** | Parallel Algorithms (Scan) | Multi-kernel pipeline |
| **9. Histogram** | **Atomics / Contention** | `atomicAdd` |
| **10. Monte Carlo** | RNG State / Real Workflow | `curand`, Host/Device reduction |

### Teaching Tips for this Level
1.  **Keep `main()` identical:** Only swap the kernel launch lines and kernel definition.
2.  **Error Checking Macro:** Give them a `CHECK(cudaGetLastError())` macro immediately; silent kernel crashes are the #1 frustration source.
3.  **NVIDIA Nsight Compute / `nvprof`:** Assign "Run Ex 7 (AoS) vs Ex 7 (SoA) in Nsight Compute" as a lab. The "Memory Throughput" metric difference is instantly visible.
4.  **Template the Boilerplate:** Provide a `cuda_util.h` with `cudaMallocCheck`, `cudaMemcpyCheck`, `initArray` so they focus purely on the kernel logic.
